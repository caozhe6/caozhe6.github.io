nginx安装与配置

发表于 2019-10-10 更新于 2019-10-21
  • Nginx 是一个高性能的http和反向代理服务器,具有占用内存少,并发能力强的优秀的性能
  • Nginx在开发中主要用作反向代理,负载均衡,动静分离

Nginx安装

  • 下载Nginx的安装包

    • wget http://nginx.org/download/nginx-1.6.2.tar.gz

  • 安装pcre

    • wget https://netix.dl.sourceforge.net/project/pcre/pcre/8.40/pcre-8.40.tar.gz
    • tar -xvf pcre…..
    • 进入目录执行./configure
    • make && make install
    • 查看版本号 pcre-config –version

  • 安装Nginx的相关依赖

    • 安装openssl,zlib
    • yum -y install make zlib zlib-devel openssl openssl-devel

  • 安装Nginx

    • 解压
    • 执行
    • make && make install

  • 安装完成之后安装在了usr/local/nginx/sbin/下

    • 启动后如果不能访问可能是防火墙限制了访问端口,解决方法如下

      • 查看开放端口:firewall-cmd –list-all

      • 设置开放端口:firewall-cmd –add-service=http –permanent

        sudo firewall-cmd –add-port=80/tcp –permanent

      • 重启防火墙:firewall-cmd –reload

  • 常用命令

    • 启动:./nginx
    • 查看版本号:./nginx -v
    • 关闭:./nginx -s stop
    • 重新加载:./nginx -s reload

配置文件

  • nginx 配置文件是/usr/local/nginx/conf/中的nginx.conf

  • nginx配置文件组成

    • 全局块:从配置文件开始到events之间的内容,设置一些影响nginx服务器整体运行的配置指令
      • worker_processes:值越大可以支持的并发处理量越高
    • events块:主要影响nginx服务器与用户的网络连接
      • worker_connections:支持的最大连接数
    • http块
      • http全局块
      • server块

  • 反向代理

    • 反向代理接收客户端请求,转发请求,这样可以隐藏webapp的真实ip更加安全

    • 在location中加入proxy_pass http://xxx.xxx.xxx.xxx:port;

  • 负载均衡

    • 负载均衡可以在高并发的情况下实现多台服务器的分流,将请求分发到多个服务器上

    • 在http节点下加入upstream节点

      • weight分配访问的权重,不写则默认轮询
      • ip_hash可以按每个访问ip的hash分配服务器;这样同一ip就会在同一服务器上访问,解决了session的问题,不写则默认轮询
      • fair根据服务器响应时间来分配

    • 在location中加入:proxy_pass http://myserver;

  • 动静分离

    • 把静态,动态资源分开,Nginx来调用动态和静态资源,可以减少webAPP的访问压力,让webAPP只处理动态资源
